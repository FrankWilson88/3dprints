Summary

Setting up the Ender-3 after it’s been sitting in the closet for about a year. The firmware was wiped in a catastrophic boating accident.

Steps Taken

  1. Find the firmware needed for the 3d printer. In my case the Ender-3. The firmware can be found on the manufacturers website. Mine is here. We’re gonna go with the factory firmware.

[!NOTE]
You need to put the URL in quotes since the geniuses over at Creality decided to use spaces in the filename. Or insert %20

    $ wget "https://file2-cdn.creality.com/file/b7da70b1c1352a5ca1eb40b83e7e3c0e/Ender-3  Marlin2.0.6 Factory firmware.zip"
  1. Unzip the file
    $ unzip "Ender-3  Marlin2.0.6 Factory firmware.zip"
  1. Mount and copy to SD Card
    $ sudo mount /dev/sd* /mnt
    # For some reason there are 2 spaces between Ender-3__Marlin* (Because China)
    $ sudo cp Downloads/"Ender-3  Marlin2.0.6 Factory firmware
Ender-3Marlin2.0.6HW4.2.2.bin" /mnt
    $ sudo umount /dev/sd*

  1. Insert the card into the 3d printer, and turn it on. The printer will auto detect the .bin file, and install the firmware.

  2. Turn off the printer, remove the SD Card, and turn the printer back on to ensure the firmware is installed correctly.

  3. For a beginner test print, we will use this website as our slicer program. I used this XYZ 20mm Calibration Cube as a test. Download the .stl.

  4. Open the webpage above. Hover over files and click import. Hover over start click slice, then export. You should get a .gcode file.

  5. Mount the SD Card again. Remove the .bin file. Copy the .gcode file to the SD Card. Unmount the SD Card

    $ sudo mount /dev/sd* /mnt
    $ sudo rm /mnt/*
    $ sudo cp ~/Downloads/*.gcode /mnt
    $ sudo umount /dev/sd*

  1. Insert the SD Card into the printer. Level the bed, as explained here.

  2. Select Print from Card on the printer. It will start the print. Watch the first few layers for inconsistencies, and either adjust or stop the print and re-level the bed.

Observations & Notes

  • I printed the bottom half of RPi case. The spacers are messy, and it’s stringy.
  • I’d like to build a server rack.

Problems / Challenges

  • On the first test prints, the first layers were turning out fine. But as it went up it would make a loud knocking sound, and print would start to run off to the side. This told me the z-axis was off. Sure enough the tension screws holding the rod in place were so lose they were almost falling out. I tightened the screws and checked to make sure it was center. it was about 2 degrees off center. I adjusted to make it plumb, and each print was fine after that.

Next Actions

  • Learn to make minor adjustments
  • Print something more advanced
  • Learn Blender and Slic3r.

Steps Taken

  1. Found this tutorial here
  2. Generated this blog
  3. Create my first square.
    > [!NOTE]
    > This should set the Home coordinates to x0y0z0(expressed as x,y,z not xN yN zN), enable absolute mode, set the hotend to 0,0,0. Heat the bed to 60 degrees F. Heat the extruder to 200 degrees F. Set the extrusion rate to 0, and set the home axis to 0,0 This would actually make a right angle that is 20 inches by 20 inches. This is wrong.
    G28 ; home the cnc machine. (i.e. move until you hit the sensors)
    G20 ; This line tells the computer to use inches not mm (G20/G21)
    G90 ; This tells the computer to measure each distance relative to the 0,0 position. It's called absolute positioning. There is relative positioning as well.
    M82 ; this line tells the extruder itself to start in absolute positioning as well.
    M140 S60 ; this section tells the bed and hotend to heat up to temp, and stay at the temp to follow further instructions. (This line is the hotend)
    M104 S200 ; This line heats up the bed
    M109 S60 ; This line says to keep the hotend at this temp, and continue with new commands
    M190 S200 ; This line says to keep the bed at this temp, and continue with new commands
    G92 E0 ; This sets the current position and extruder to 0,0,0 (essentially, set the bed, set the extruder, define the x,y,z axis, so the bed is x,y the extruder is z, and set the extrusion rate to 0)
    G0 X0 Y0 ; this is telling the computer, "where you are is the default 0,0,0 spot. regardless of everything else."
    G1 Z0.2 F1200 ; this is telling the extruder to move to 0.2 mm above the bed with a feed rate of 1200 mm/min. remember, this could change. for this line imagine auto bed leveling, it's saying this is the distance from the bed i need to start feeding the extruder. So if a piece of paper is 0.2mm, add 0.2mm from that, and this is where we start to print. if your printer line beads up, change this line.
    G1 X10 Y10 F3000 ; (my first mistake) this sets the default coordinate at 10 inches by 10 inches on the bed. with absolute coordinates, it draws a line from 0,0 to 10,10. we don't want that. because we want the base of the object at 10,10. the problem is, your bed is only 9x9 and 8x8 of usable space. So it shoots the design off into the void, and draws a line from your original home (G28). into another void that ends no where, because the end is not defined.
    G1 E6 F200
    G1 X20 Y10 E10 F3000
    G1 X20 Y20 E10 F3000
    G1 X-20 Y20 E10 F3000 ; at this point the line should shoot off into no where (the void) because we assume we want to subtract 20 inches to return to zero (the 10 spot), but infact, it sends us to the negative 20 x axis. so if the corner we started at was 0,0, we tell the printer to go to -20,0. this shoots the line we run; into a spot that the cad software will not zoom to, or maybe, bu it won't make sense.
    G1 X-20 Y-20 E10 F3000
    ; Copy and past all of this text to Notepad, into a slicer program, and you will see that is does not throw an error from all these comments, but it makes some weird lines.
  1. I was using the slicer to see what the machine was assuming i meant. and to see adding comments, which didn’t throw an error. When I changed the settings to mm(G21) it readjusted a lot better. Not because the code was “wrong” but because all cad software measures in mm. so when i measure in inches the cad software adjusts, but since my math sucks its not exact, and it throws the equations into the void. so this is mm not inches.
    G28 ; We determined the home coordinates here.
    G21; Tell the machine to use mm
    G90
    M82
    M140 S60
    M104 S200
    M109 S60
    M190 S200
    G92 E0
    G0 X0 Y0 ; notice this is the starting point. so it should draw a line from 0,0 to 100,100. this is where my assumptions start.
    G1 Z0.2 F1200 ; this tells the printer to move the z axis to 0.2, or, 0.2mm above where you leveled the bed.
    G1 X100 Y100 F3000 ; this is where it draws a line to the 100,100 spot. Notice there is not extrusion rate, therefore it will not feed filament to the starting corner of the box.
    G1 E6 F200 ; at this point we set the extrusion rate to 6. we can set it to a lower level, so that later extrusions do not cause collisions. remember, the extrusion rates is compared to the feed rate.
    G1 X125 Y100 E10 F3000 ; we move the extruder and bed to match an axis of x,y,z along the way, the extruder should send 4 more mm of filament, at a rate of 3000 mm per minute.
    G1 X125 Y125 E10 F3000 ; at this point, the extruder will move in a right angle, but the extruder will not output more filament, because you are telling it to feed the same 10mm at a rate of 3000 mm/min. it will not feed more filament, because the extrusion is stuck at 10!
    G1 X100 Y125 E10 F3000 ; this will finish the left size  of the box with no extrusion.
    G1 X100 Y100 E10 F3000 ; this will bring it back to it's original (although off from the start) 0,0 point, without actually printing anything.
    ; So if you imagine the filament like a ruler, It extruded to 10 inches, and then stayed at 10 inches. So we used 10 inches of filament for the first side, and then stopped.
    ; This will draw a box but not the infill.
